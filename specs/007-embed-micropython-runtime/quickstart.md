# Quickstart: Embed MicroPython Runtime

**Feature**: 007-embed-micropython-runtime

Two developer paths. Choose one.

---

## Firmware Developer

You modify C++, add C modules, or change the build system.

### First-time setup

```bash
cd firmware/Firmware-0308-modular/
cp BadgeConfig.h.example BadgeConfig.h   # fill in WiFi + server URL
./setup.sh                                # installs arduino-cli, libraries,
                                          # downloads MicroPython v1.27.0,
                                          # generates micropython_embed/
```

### Build and flash

```bash
./build.sh -n    # compile firmware + pack apps into LittleFS image
./flash.py       # flash firmware then apps (two separate esptool steps)
```

Flags for partial flashing:
```bash
./flash.py --firmware   # firmware only (skips apps)
./flash.py --apps       # apps only (skips firmware — fast iteration on Python changes)
./flash.py --build      # recompile then flash everything
```

### Adding a function to the `badge` module

1. Edit `badge_module.c` — add your `MP_DEFINE_CONST_FUN_OBJ_*` entry and
   add it to `badge_module_globals_table`.
2. Re-run the embed generation step (QSTR regeneration required):
   ```bash
   ./setup.sh --regen-embed
   ```
3. `./build.sh -n` — verify it compiles.
4. Update `contracts/badge-module-api.md` with the new function.

### Tuning heap size

In `BadgeConfig.h`:
```c
#define MICROPY_HEAP_SIZE   (128 * 1024)  // bytes; default 128 KB
#define MP_SCRIPT_MAX_BYTES 16384         // bytes; max .py source file size
```

## Adding MicroPython Libraries

There are four ways to make additional Python functionality available to badge apps,
in order from simplest to most complex.

---

### Method 1 — Enable a built-in MicroPython module

MicroPython ships many standard modules that are off by default to save flash.
Turn one on by editing `mpconfigport.h` and re-running embed generation.

**Currently enabled** (see data-model.md section 5 for the full flag table):
`utime`, `math`, `ujson`

**Example: enable `ustruct`**

1. Open `firmware/Firmware-0308-modular/badge_mp/mpconfigport.h`
2. Add or set: `#define MICROPY_PY_USTRUCT 1`
3. Regenerate and rebuild:
   ```bash
   ./setup.sh --regen-embed
   ./build.sh -n
   ```
4. Apps can now `import ustruct`.

**Common flags** (set to `1` to enable):

| Flag | Module | Use |
|------|--------|-----|
| `MICROPY_PY_USTRUCT` | `ustruct` | Binary data packing |
| `MICROPY_PY_UCOLLECTIONS` | `ucollections` | `OrderedDict`, `namedtuple` |
| `MICROPY_PY_URE` | `ure` | Regex |
| `MICROPY_PY_UIO` | `uio` | In-memory streams |
| `MICROPY_PY_UHEAPQ` | `uheapq` | Heap queue |
| `MICROPY_PY_URANDOM` | `urandom` | Random numbers |

Full flag reference: `micropython/py/mpconfig.h` (generated by `setup.sh`).

---

### Method 2 — Add a frozen pure-Python library

Frozen modules are pure-Python `.py` files compiled into the firmware binary at build
time. They are available to all apps via `import` with no VFS files needed.

**Use this for**: shared utility libraries (e.g. a debounce helper, an animation
library, a third-party pure-Python package from micropython-lib).

**Step-by-step**:

1. Add your `.py` file to `firmware/Firmware-0308-modular/freeze/`:
   ```bash
   cp my_utils.py firmware/Firmware-0308-modular/freeze/
   ```

2. Register it in `firmware/Firmware-0308-modular/freeze/manifest.py`:
   ```python
   freeze("$(PORT_DIR)/freeze/my_utils.py")
   ```

3. Regenerate and rebuild:
   ```bash
   ./setup.sh --regen-embed
   ./build.sh -n
   ```

4. Apps can now `import my_utils`.

---

### Method 3 — Add a C extension module

Use this for hardware peripherals, performance-critical code, or anything that needs
direct access to ESP32 APIs or existing C++ firmware modules.

**Step-by-step**:

1. Create `firmware/Firmware-0308-modular/badge_xxx_module.c`:
   ```c
   #include "py/runtime.h"
   #include "py/obj.h"

   static mp_obj_t badge_xxx_hello(void) {
       return mp_obj_new_str("hello from C", 12);
   }
   static MP_DEFINE_CONST_FUN_OBJ_0(badge_xxx_hello_obj, badge_xxx_hello);

   static const mp_rom_map_elem_t badge_xxx_module_globals_table[] = {
       { MP_ROM_QSTR(MP_QSTR___name__), MP_ROM_QSTR(MP_QSTR_badge_xxx) },
       { MP_ROM_QSTR(MP_QSTR_hello),    MP_ROM_PTR(&badge_xxx_hello_obj) },
   };
   static MP_DEFINE_CONST_DICT(badge_xxx_module_globals, badge_xxx_module_globals_table);

   const mp_obj_module_t badge_xxx_module = {
       .base = { &mp_type_module },
       .globals = (mp_obj_dict_t *)&badge_xxx_module_globals,
   };
   MP_REGISTER_MODULE(MP_QSTR_badge_xxx, badge_xxx_module);
   ```

2. Regenerate embed (QSTR scanner must see the new `.c` file):
   ```bash
   ./setup.sh --regen-embed
   ./build.sh -n
   ```

3. Apps can now `import badge_xxx`.

> **Key rule**: The `.c` file with `MP_REGISTER_MODULE` must exist **before**
> running `setup.sh --regen-embed`. The QSTR/moduledefs scanner reads all `.c`
> files in the sketch directory.

---

### Method 4 — Bundle a library inside your app file

For small, self-contained utilities that only one app needs, paste the library code
directly into your `.py` file above your app logic.

No build changes needed. Works within the 16 KB source file size limit.

---

### Which method to use?

| Situation | Method |
|-----------|--------|
| A MicroPython standard library module (struct, re, random...) | 1 — enable built-in flag |
| A shared utility all apps will use (animation, debounce, formatting) | 2 — frozen module |
| Hardware access, performance, or ESP32 API wrapping | 3 — C extension |
| A small helper used by one app only | 4 — inline in app file |

## App Developer

You write Python apps that run on the badge. You do **not** need the Arduino toolchain —
just a text editor. Apps are plain `.py` files dropped into `apps/`.

### Your first app

Create `apps/my_app.py`:

```python
import badge
import utime

badge.display.clear()
badge.display.text("Hello!", 40, 32)
badge.display.show()

utime.sleep_ms(2000)
badge.exit()
```

Get it onto a badge: drop the file in `apps/`, then the badge owner runs `./build.sh -n && ./flash.py --apps`.

Run it: **Menu → Apps → my_app**.

---

### MicroPython basics

The badge runs [MicroPython v1.27.0](https://docs.micropython.org/en/v1.27.0/) — a lean
Python 3 implementation for microcontrollers. The language is Python 3, but the standard
library is smaller. Everything below is standard Python syntax.

**Variables and types**

```python
x = 42           # int
name = "Temporal" # str
ratio = 0.5      # float
flag = True      # bool
items = [1, 2, 3] # list
```

**Conditionals**

```python
if badge.button_pressed(badge.BTN_UP):
    badge.display.text("UP!", 0, 32)
elif badge.button_pressed(badge.BTN_DOWN):
    badge.display.text("DOWN!", 0, 32)
else:
    badge.display.text("...", 0, 32)
```

**Loops**

```python
# Loop forever until a button is pressed
while not badge.button_pressed(badge.BTN_RIGHT):
    utime.sleep_ms(20)

# Loop with a counter
for i in range(5):
    badge.display.text(str(i), 0, 32)
    badge.display.show()
    utime.sleep_ms(500)
```

**Functions**

```python
def show_message(msg, delay_ms=1000):
    badge.display.clear()
    badge.display.text(msg, 0, 32)
    badge.display.show()
    utime.sleep_ms(delay_ms)

show_message("Hello!")
show_message("Bye!", delay_ms=500)
```

**Exception handling**

```python
try:
    result = badge.http_get("https://example.com/api")
except OSError as e:
    badge.display.text("Network error", 0, 32)
    badge.display.show()
    utime.sleep_ms(1500)
```

**Further reading**: [MicroPython documentation](https://docs.micropython.org/en/v1.27.0/) —
the language reference, library reference, and quick reference for ESP32 all apply.

### The display

The OLED is 128×64 pixels. (0, 0) is the top-left corner. The font is 6×10 px — roughly
21 characters wide and 6 lines tall.

```python
badge.display.clear()                       # wipe the buffer
badge.display.text("Line 1", 0, 10)        # x=0, y=10 (baseline)
badge.display.text("Line 2", 0, 24)        # next line down
badge.display.text("Centered-ish", 16, 38)
badge.display.show()                        # flush buffer → OLED
```

> `show()` is the only call that touches the physical display. Call `clear()` + draw
> everything + `show()` as one atomic update to avoid flickering.

**Layout planning** (6×10 font, y = baseline):

| y | Row |
|---|-----|
| 10 | Row 1 |
| 24 | Row 2 |
| 38 | Row 3 |
| 52 | Row 4 |

### Buttons

Buttons return `True` when physically pressed (active-low GPIO, debounce is your
responsibility).

```python
import badge, utime

# Polling loop — check every 20ms
while not badge.button_pressed(badge.BTN_UP):
    utime.sleep_ms(20)
```

**Button constants** (XIAO ESP32-S3 GPIO numbers):

| Constant | GPIO |
|----------|------|
| `badge.BTN_UP` | 44 |
| `badge.BTN_DOWN` | 7 |
| `badge.BTN_LEFT` | 8 |
| `badge.BTN_RIGHT` | 9 |

**Simple debounce pattern**:

```python
def wait_for_press(btn, hold_ms=50):
    """Block until button is held for hold_ms to debounce."""
    while True:
        if badge.button_pressed(btn):
            utime.sleep_ms(hold_ms)
            if badge.button_pressed(btn):
                return
        utime.sleep_ms(10)
```

### Joystick

The joystick uses two ADC channels. Values are 0–4095 (12-bit). Centre ≈ 2048,
but varies — read the resting value and subtract to get a relative offset.

```python
raw_x = badge.adc_read(badge.JOY_X)   # 0–4095
raw_y = badge.adc_read(badge.JOY_Y)

# Normalise to -1.0 … +1.0
norm_x = (raw_x - 2048) / 2048.0
norm_y = (raw_y - 2048) / 2048.0

# Apply a deadband to ignore drift near centre
DEADBAND = 0.08
if abs(norm_x) < DEADBAND:
    norm_x = 0.0
if abs(norm_y) < DEADBAND:
    norm_y = 0.0
```

### Time

```python
import utime

utime.sleep_ms(500)             # pause 500ms
utime.sleep(1)                  # pause 1 second (float ok)

start = utime.ticks_ms()
# ... do work ...
elapsed = utime.ticks_diff(utime.ticks_ms(), start)
badge.display.text(str(elapsed) + "ms", 0, 32)
```

> `ticks_ms()` wraps after ~12 days. Use `ticks_diff(end, start)` (not subtraction)
> to handle the wraparound correctly.
>
> Reference: [utime module docs](https://docs.micropython.org/en/v1.27.0/library/time.html)

### HTTP requests

WiFi is already connected when your app runs (the firmware connects at boot).

```python
import badge, ujson

# GET
try:
    body = badge.http_get("https://example.com/api/hello")
    data = ujson.loads(body)
    badge.display.text(data["message"][:20], 0, 32)
except OSError:
    badge.display.text("request failed", 0, 32)
badge.display.show()

# POST
payload = ujson.dumps({"badge_id": "abc123"})
try:
    resp = badge.http_post("https://example.com/api/boop", payload)
except OSError:
    pass
```

Both functions block until complete. `Content-Type: application/json` is set automatically on POST.

**Badge identity** — use `badge.uid()` and `badge.server_url()` to construct URLs against the
badge's own server without hardcoding credentials:

```python
uid = badge.uid()                          # e.g. "a1b2c3d4e5f6"
server = badge.server_url()               # e.g. "https://badge.example.com"
url = server + "/api/v1/badge/" + uid + "/info"
body = badge.http_get(url)
```

> Reference: [ujson module docs](https://docs.micropython.org/en/v1.27.0/library/json.html)

### IR

The IR ISR runs continuously in the background on Core 0 and buffers received frames.
Poll from Python with `ir_available()` / `ir_read()`.

```python
import badge, utime

# Send
badge.ir_send(badge.ROLE_ATTENDEE, 0x01)

# Receive — poll in a loop
while True:
    if badge.ir_available():
        frame = badge.ir_read()
        if frame:
            addr, cmd = frame
            badge.display.clear()
            badge.display.text(f"IR: {addr:#x} {cmd:#x}", 0, 32)
            badge.display.show()
    utime.sleep_ms(50)   # poll at most every 50ms to avoid buffer overflow
```

> **Poll within 50ms** — the IRremote ring buffer holds 8 frames. Polling slower
> than 50ms risks overflow if frames arrive rapidly.

### GC (garbage collection)

MicroPython's GC runs automatically, but can pause for up to ~20ms on a 128 KB heap.
For smooth animations or timing-sensitive code, call `badge.gc_collect()` at a known
safe point (e.g. once per frame, before a `sleep_ms`).

```python
while True:
    badge.gc_collect()       # force GC now, at a predictable point
    badge.display.clear()
    # ... draw frame ...
    badge.display.show()
    utime.sleep_ms(33)       # ~30fps
```

### Exiting

All apps should exit via `badge.exit()` or by reaching the end of the script. Both
return cleanly to the main menu.

```python
badge.display.clear()
badge.display.text("Done!", 40, 32)
badge.display.show()
utime.sleep_ms(1000)
badge.exit()
# or just fall off the end of the script — same effect
```

**Emergency exit**: hold **BTN_UP + BTN_DOWN** simultaneously for ≥50ms from anywhere
in the app. The badge returns to the menu within 1 second. No app code needed.

### Common patterns

**Menu inside an app**

```python
import badge, utime

options = ["Option A", "Option B", "Option C"]
cursor = 0

while True:
    badge.display.clear()
    for i, opt in enumerate(options):
        prefix = "> " if i == cursor else "  "
        badge.display.text(prefix + opt, 0, 12 + i * 14)
    badge.display.text("UP/DN=move  RT=select", 0, 58)
    badge.display.show()

    while True:
        if badge.button_pressed(badge.BTN_UP):
            cursor = (cursor - 1) % len(options)
            utime.sleep_ms(200)
            break
        if badge.button_pressed(badge.BTN_DOWN):
            cursor = (cursor + 1) % len(options)
            utime.sleep_ms(200)
            break
        if badge.button_pressed(badge.BTN_RIGHT):
            # handle selection
            badge.display.clear()
            badge.display.text("Selected: " + options[cursor], 0, 32)
            badge.display.show()
            utime.sleep_ms(1000)
            break
        if badge.button_pressed(badge.BTN_LEFT):
            badge.exit()
        utime.sleep_ms(20)
```

**Countdown timer**

```python
import badge, utime

for secs in range(10, 0, -1):
    badge.display.clear()
    badge.display.text("Time left:", 20, 20)
    badge.display.text(str(secs) + "s", 52, 40)
    badge.display.show()
    utime.sleep_ms(1000)

badge.display.clear()
badge.display.text("Time's up!", 20, 32)
badge.display.show()
utime.sleep_ms(1500)
badge.exit()
```

**Reading joystick direction**

```python
import badge, utime

DEADBAND = 0.3

def joystick_dir():
    x = (badge.adc_read(badge.JOY_X) - 2048) / 2048.0
    y = (badge.adc_read(badge.JOY_Y) - 2048) / 2048.0
    if y < -DEADBAND: return "UP"
    if y >  DEADBAND: return "DOWN"
    if x < -DEADBAND: return "LEFT"
    if x >  DEADBAND: return "RIGHT"
    return None

while True:
    d = joystick_dir()
    badge.display.clear()
    badge.display.text(d or "centre", 30, 32)
    badge.display.show()
    utime.sleep_ms(80)
```

## File size limits

| Limit | Default |
|-------|---------|
| Max `.py` file size | 16 384 bytes (16 KB) |
| MicroPython heap | 131 072 bytes (128 KB) |

Large apps that allocate many objects may run out of heap (`MemoryError`). The badge
will catch this and return to the menu. Keep v1 apps lightweight.
